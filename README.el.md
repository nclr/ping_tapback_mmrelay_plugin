# ping_tapback_mmrelay_plugin

[🇬🇧 English](README.md) | 🇬🇷 Ελληνικά

Ένα προσαρμοσμένο plugin για το [meshtastic-matrix-relay](https://github.com/jeremiah-k/meshtastic-matrix-relay) που αντιδρά σε μηνύματα Meshtastic με emoji tapbacks και υποστηρίζει την εκτέλεση traceroutes από το Matrix.

## Λειτουργίες

### Tapback με μέτρηση hops

Όταν ένας κόμβος στείλει μήνυμα που περιέχει μια λέξη-έναυσμα, το plugin απαντά με ένα emoji tapback που δείχνει πόσα hops διένυσε το μήνυμα.

| Λέξεις-έναυσμα | Απάντηση |
|---|---|
| `ping`, `test`, `hello` | emoji μέτρησης hops (0️⃣–8️⃣) |
| `πίνγκ`, `τέστ`, `τεστ` (Ελληνικά) | emoji μέτρησης hops |
| `καλημέρα` / `kalimera` | ☀️ |
| `καλησπέρα` / `good afternoon` | 🌞 |
| `καληνύχτα` / `kalinixta` / `good night` | 🌙 |

Το tapback αποστέλλεται ως **αντίδραση** στο αρχικό μήνυμα (χρησιμοποιώντας `reply_id` + `emoji=1`), οπότε εμφανίζεται ως φούσκα αντίδρασης στους Meshtastic clients που το υποστηρίζουν, αντί για νέο μήνυμα.

### Traceroute από το Matrix

Στείλτε `!traceroute <κόμβος>` σε ένα δωμάτιο Matrix για να ενεργοποιήσετε ένα traceroute προς οποιονδήποτε κόμβο του mesh. Μπορείτε να περάσετε είτε ένα hex node ID (`!abcd1234`) είτε ένα σύντομο όνομα, το οποίο αναλύεται με βάση την τρέχουσα λίστα κόμβων. Το plugin απαντά με την πλήρη διαδρομή και τις τιμές SNR για κάθε hop, και προς τις δύο κατευθύνσεις.

**Παράδειγμα:**
```
!traceroute !abcd1234
!traceroute MN1
```

Πριν την εκτέλεση, επιβεβαιώνει με μια περιγραφική γραμμή κατάστασης που περιλαμβάνει το πλήρες όνομα, το σύντομο όνομα και το ID του στόχου. Όταν ο κόμβος έχει γνωστή τελευταία θέση και/ή ακούστηκε πρόσφατα, προσθέτει επίσης έναν σύνδεσμο OpenStreetMap προς τις συντεταγμένες του και μια χρονοσφραγίδα «last heard»:
```
🔍 Running traceroute to MyNode (MN1) `!abcd1234`…
📍 37.97640, 23.72784 — [OpenStreetMap](https://www.openstreetmap.org/?mlat=37.97640&mlon=23.72784#map=15/37.97640/23.72784)
🕒 Last heard: 2026-06-06 13:37 UTC (12m ago)
```
Οι γραμμές 📍 και 🕒 περιλαμβάνονται μόνο όταν τα δεδομένα αυτά είναι διαθέσιμα για τον κόμβο· διαφορετικά το μήνυμα κατάστασης περιορίζεται μόνο στην πρώτη γραμμή.

**Απάντηση:**
```
Towards destination: !relay → !abc123 (1.5 dB) → !abcd1234 (3.25 dB)

Back to us: !abcd1234 → !abc123 (1.75 dB) → !relay (2.5 dB)
```

Τα hops χωρίς δεδομένα SNR εμφανίζονται ως `? dB (unknown SNR)`.

Το traceroute εκτελείται στο παρασκήνιο ώστε το plugin να παραμένει αποκρίσιμο σε άλλες εντολές όσο περιμένει το αποτέλεσμα (έως χρονικό όριο 90 δευτερολέπτων).

**Διαχείριση πολλαπλών hops:** το υποκείμενο callback `onResponse` του meshtastic είναι ένας handler μίας χρήσης που καταναλώνεται από το πρώτο πακέτο που αναφέρεται στο αίτημα — συνήθως ένα πρώιμο ack `ROUTING_APP` σε διαδρομές πολλαπλών hops — κάτι που προηγουμένως προκαλούσε την απώλεια απαντήσεων από κόμβους που απέχουν περισσότερο από ένα hop. Για να παρακαμφθεί αυτό, το plugin ακούει επίσης στον γενικό δίαυλο λήψης του meshtastic και αντιστοιχίζει τη γνήσια απάντηση `TRACEROUTE_APP` βάσει portnum και κόμβου προέλευσης, ώστε τα traceroutes πολλαπλών hops να αναφέρονται αξιόπιστα.

## Εγκατάσταση

Τοποθετήστε τον φάκελο του plugin στον κατάλογο custom plugins του mmrelay (συνήθως `~/.mmrelay/plugins/custom/ping_tapback/`) και ενεργοποιήστε το στη διαμόρφωση του mmrelay:

```yaml
custom-plugins:
  ping_tapback:
    active: true
```

## Διαμόρφωση

Το plugin σέβεται τις τυπικές ρυθμίσεις καναλιού και άμεσων μηνυμάτων του mmrelay. Ενεργοποιήστε το ανά κανάλι στη διαμόρφωση `matrix_rooms`, όπως κάθε άλλο plugin.
